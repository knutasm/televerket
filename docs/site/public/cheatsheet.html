dbt cheat sheet

Vanlige kommandoer og flagg

Select-syntax

kan kombineres i de fleste kontekster
Velg modeller
dbt run Kjør alle modeller i prosjektet
dbt run --select stg_crm__customers Kjør én spesifikk modell
dbt run --select staging Kjør alle modeller i en mappe
dbt run --select tag:daily Velg modeller med et bestemt tag
Grafsyntaks — avhengigheter
dbt run --select +mart_customer_360 Modellen og alle oppstrøms-avhengigheter
dbt run --select mart_customer_360+ Modellen og alle nedstrøms-avhengigheter
dbt run --select +mart_customer_360+ Hele grafen i begge retninger
dbt run --select 1+stg_crm__customers Maks 1 nivå oppstrøms
dbt run --select stg_crm__customers+1 Maks 1 nivå nedstrøms
Kombinere og ekskludere
dbt run --select staging intermediate Mellomrom = union (begge)
dbt run --select +mart_customer_360 --exclude staging Ekskluder en gruppe fra et bredere utvalg
dbt run --select source:crm+ Alt nedstrøms fra en bestemt source
--select / -s fungerer likt på tvers av dbt run, dbt test, dbt build og dbt compile — velger alltid samme sett med noder.

Sources

Grunnleggende
dbt source freshness Sjekk freshness på alle registrerte kilder
dbt source freshness --select source:crm Freshness kun for én source-gruppe
dbt source freshness -o freshness.json Skriv resultatet til fil
Grafsyntaks for sources
dbt run -s source:billing Direkte avhengig av en source-gruppe
dbt run -s source:billing+ Alle nedstrøms-modeller fra source
dbt run -s source:billing.billing_invoices+ Nedstrøms fra én spesifikk kildetabell
I SQL — referanseformat
{{ source('crm', 'crm_customers') }} Referer til en kildetabell. Brukes kun i staging-modeller.
{{ ref('stg_crm__customers') }} Referer til en annen dbt-modell. Brukes overalt unntatt mot kilden.
{{ ref('other_project', 'mart_customers') }} Cross-project ref — referer til modell i et annet dbt-prosjekt.
ref() og source() er ikke bare referanser — de bygger linjeavstamningsgrafen som dbt bruker til --select-syntaks, docs og CI. Hardkodede tabellnavn bryter grafen.

Testing

Kjøre tester
dbt test Kjør alle tester i prosjektet
dbt test -s stg_crm__customers Tester for én modell
dbt test -s staging Alle tester i en mappe
dbt test -s source:crm Tester (inkl. freshness) for en source
dbt test --store-failures Skriv feilende rader til tabeller i DB
Innebygde tester — full YAML-syntaks
- not_null Ingen null-verdier i kolonnen
- unique Alle verdier er unike
- accepted_values: values: ['active', 'churned', 'suspended'] # quote: false → sammenlign uten quotes Verdier må finnes i listen. Feiler på alt som ikke er listet — husk å inkludere alle gyldige statuser.
- relationships: to: ref('stg_crm__customers') field: customer_id # field = PK i den andre modellen Hver verdi i kolonnen må finnes som customer_id i stg_crm__customers. Brukes på fremmednøkler. Null-verdier hoppes over som standard.
Kombinere med build
dbt build Kjør run + test + seed + snapshot i rekkefølge
dbt build -s +mart_customer_360 Build hele grafen med tester underveis

Dokumentasjon

Generer og vis
dbt docs generate Bygg dokumentasjonssiden fra YAML og skjema
dbt docs serve Åpne dokumentasjonssiden i nettleseren
Sjekk dekning
dbt compile Valider at alle modeller kompilerer uten å kjøre dem
dbt parse Les og valider hele prosjektet (raskere enn compile)
Doc-blokker i markdown
{% docs customer_id %}
...{% enddocs %} Definer gjenbrukbar beskrivelse i en .md-fil
description: '{{ doc("customer_id") }}' Bruk doc-blokken i YAML

Nyttige flagg

fungerer på de fleste kommandoer
--select / -svelg noder
--excludeekskluder noder
--target / -tbytt miljø (dev/prod)
--profiles-dirangi profiles.yml-sti
--varssend variabler inn i run
--full-refreshtving rebuilding av inkrementelle modeller
--store-failureslagre feilrader i DB
--no-partial-parsetving full parsing av prosjektet
--threads Nantall parallelle tråder
--warn-errorgjør advarsler til feil
--deferbruk prod-manifestet for ikke-valgte noder
--favor-statebruk prod-artefakter der det er mulig
